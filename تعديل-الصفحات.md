خوارزمية `_hijriNow()` كانت تستخدم Kuwaiti/civil algorithm — تعطي نتائج متأخّرة 2-3 أيام عن `HijriDate.getToday()` في `js/hijri-date.js`.

**الإصلاح**: استبدال الخوارزميّة بنفس معادلات `js/hijri-date.js` بالضبط:

النتيجة: **SSR = JS** تماماً (لا مزيد من الاختلاف بين ما يراه الـ crawler والـ user).

#### ما تغيّر

**1. `_cached(key, fetchFn, ttlMs)` helper جديد** في `js/app.js` (بعد `nomUrl`):
```js
async function _cached(key, fetchFn, ttlMs) {
    try {
        const raw = localStorage.getItem(key);
        if (raw) {
            const parsed = JSON.parse(raw);
            if (parsed && parsed.v !== null && (Date.now() - parsed.t) < ttlMs) {
                return parsed.v;   // cache hit — فوري
            }
        }
    } catch(e) {}
    const value = await fetchFn();
    if (value !== null && value !== undefined) {
        try { localStorage.setItem(key, JSON.stringify({ v: value, t: Date.now() })); } catch(e) {}
    }
    return value;
}
function _coordKey(prefix, lat, lng, lang) {
    // 4 منازل ≈ دقة 11 متر (كافٍ لتحديد مدينة)
    const la = Math.round(parseFloat(lat) * 10000) / 10000;
    const lo = Math.round(parseFloat(lng) * 10000) / 10000;
    return `${prefix}_${la}_${lo}${lang ? '_' + lang : ''}`;
}
```
